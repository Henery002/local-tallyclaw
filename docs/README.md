# 文档中心

本目录是 TallyClaw 的长期项目记忆。

架构说明、决策记录、迭代日志、排障总结和交接材料都应沉淀在这里。临时笔记不要放在仓库根目录。

## 目录说明

- `architecture/`：当前系统设计、数据流、持久化模型和技术边界。
- `decisions/`：长期有效的架构决策记录。
- `guides/`：面向人类开发者和 AI Agent 的接入与操作指南。
- `records/`：按时间沉淀的讨论总结、迭代日志、长期运维状态、问题报告和修复记录。

## 记录规则

后续每一次功能迭代、重大方案讨论、重要问题修复或项目方向变化，都应至少更新一份 `docs/records/` 下的记录文件；如果变更对用户可见、影响项目结构或属于功能迭代/问题修复，也应同步更新按日期降序维护的 `CHANGELOG.md`。

记录文档同时也是长期运维文档。它们的第一读者包括后续接入的 Codex 和其他 AI Agent，因此应明确写出当前状态、已知边界、未决问题、证据路径和可执行下一步。

## 语言规范

项目文档默认使用中文维护。必要的技术名词、路径、模块名、字段名和代码标识可以保留英文。
