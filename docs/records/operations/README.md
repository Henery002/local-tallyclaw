# 长期运维记录

本目录保存 TallyClaw 的长期运维状态和 Agent 续接入口。

这里的“运维”主要指项目自身的持续开发运维：当前阶段、文档状态、实现状态、安全边界、已知风险、下一步任务和交接信息。它不表示 TallyClaw 可以治理或修改上游工具；TallyClaw 仍然只做只读观测。

## 文件说明

- `current-state.md`：当前项目状态与下一步，是新 Agent 进入项目时必须阅读的文件。

## 更新规则

每次较完整的开发、分析、修复、方案调整或重要讨论结束前，都应检查是否需要更新 `current-state.md`。

记录应优先回答：

- 现在项目处于什么阶段？
- 最近完成了什么？
- 有哪些安全边界不能突破？
- 当前最重要的风险或未决问题是什么？
- 下一个 Agent 可以直接执行什么？
- 证据和相关文档在哪里？
