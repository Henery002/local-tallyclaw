# 项目记录

本目录保存按时间沉淀的项目记忆，也是 TallyClaw 的长期运维文档区。

使用这里记录讨论、迭代、排障、当前状态和交接信息，避免后续工作难以续接，也避免在仓库根目录散落临时笔记。

本项目后续将主要通过 AI Agent 开发，因此记录文档应优先服务 Agent 续接：让新的 Agent 能快速知道项目当前状态、不能做什么、下一步做什么、证据在哪里。

## 子目录

- `discussions/`：重大设计讨论和产品决策。
- `iterations/`：功能迭代记录和实现进展。
- `fixes/`：排障总结和修复记录。
- `operations/`：长期运维状态、当前进展、风险、边界和交接入口。

## 必须更新记录的情况

出现以下情况时，应更新项目记录：

- 新增或实质变更一个功能。
- 重大设计讨论改变项目方向。
- 调查或修复了一个问题。
- 新增或修改数据源接入。
- 澄清安全边界。
- 需要为另一个 AI Agent 留下交接上下文。
- 当前项目状态、下一步或风险发生变化。

## 风格

优先使用简短、带日期的记录，并包含：

- 背景。
- 决策或变更。
- 证据路径。
- 后续事项。

## 长期运维要求

`operations/current-state.md` 是后续 Agent 的主要续接入口。每次较完整的开发、分析、修复或重大讨论结束前，都应更新它，至少确认：

- 当前项目阶段。
- 最近完成事项。
- 当前安全边界。
- 已知风险或未决问题。
- 下一步建议。
- 相关证据文档路径。

## 语言规范

记录性文档默认使用中文维护。技术标识、路径、字段名、模块名可以保留英文。
