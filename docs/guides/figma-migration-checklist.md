# Figma 设计迁移检查清单

本清单用于后续从 Figma 增量迁移 TallyClaw 宠物 UI，避免每次重新踩窗口尺寸、裁切、透明背景和动效节奏问题。它是长期开发指南，不作为按次讨论记录维护。

## 迁移前

- 确认 Figma 当前版本、节点范围和截图，避免把同包内无关代码迁入项目。
- 明确本次是全量替换还是局部增量：宠物本体、hover 小浮层、点击展开面板、动效参数、颜色或尺寸。
- 保留只读数据和桌面浮窗边界：UI 改动不应影响数据源读取、ledger 写入和系统偏好。

## 视觉检查

- 宠物本体不应有额外背景框、外圈圆形底色或不属于设计稿的窗口边框。
- 透明区域必须保持真实透明，窗口阴影只来自宠物或面板自身，而不是 NSWindow 背景。
- 宠物尺寸、hover 小浮层尺寸、点击面板尺寸需要同时检查，不能只对齐单个元素。
- 面板内长数字需要完整展示，必要时通过宽度、字重、字号或最小缩放处理，不能被省略或裁切。
- 宠物靠屏幕左、右、上、下边缘时，hover 小浮层和展开面板都不能被窗口画布裁切。

## 动效检查

- idle 状态应为极轻微、自然、顺滑的持续浮动，避免高频跳动或大幅抖动。
- activity 状态只在真实 token / request 增量后短暂触发，不能因历史总量存在而长期保持活跃色。
- hover、点击、展开面板都应有平滑过渡，点击时宠物本体保持原地，面板从宠物下方自然显现。
- 眨眼、核心呼吸、粒子等动效应互不抢节奏，整体 CPU 占用保持轻量。

## 交互检查

- 宠物可以拖到屏幕边缘，尤其是顶部；重启后应恢复上次拖拽位置。
- 常驻顶层和普通浮窗模式切换后，窗口层级应立即生效。
- hover 小浮层应根据宠物当前位置自动选择左侧或右侧，优先避开屏幕边缘。
- 点击展开面板应优先显示在宠物下方；如未来增加边缘避让，也应保持宠物本体不跳位。

## 验证建议

- 运行 `swift build` 和 `swift test`，确认 UI 改动没有破坏核心模型与数据源测试。
- 使用 `./script/build_and_run.sh --verify` 启动本地 app，手动检查透明背景、拖拽、hover、展开面板和状态色。
- 如改动涉及窗口尺寸或画布边界，至少检查屏幕左侧、右侧、顶部和底部四个位置。
- 迁移完成后只把用户可见变更简要写入 `CHANGELOG.md`，不新增过程记录文档。
