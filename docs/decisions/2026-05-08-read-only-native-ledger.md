# 决策：只读原生账本架构

日期：2026-05-08  
状态：已接受基线

## 背景

TallyClaw 需要在本机多个工具和网关之间观测 token 用量，同时保持轻量和安全。源工具可能包含账号凭据、refresh token、运行时数据库、日志和统计文件，这些内容都不能被修改。

桌面宠物还需要在 macOS 上具备流畅动画、低资源占用和可靠的浮动窗口行为。

## 决策

采用原生 macOS 架构、严格只读的数据源适配器，以及项目自有 SQLite 账本。

TallyClaw 将会：

- 从已批准的源工具和网关读取数据。
- 将记录归一化到自己的数据模型。
- 保守去重。
- 将长期计数持久化到自己的 SQLite 数据库。
- 使用原生 macOS UI 技术渲染轻量、可拖动的桌面宠物。

TallyClaw 不会：

- 写入源工具文件。
- 刷新源账号凭据。
- 清理、修复、压缩或迁移源数据。
- 在完成观测记录后继续依赖源文件保存永久总量。

## 影响

该选择让项目更安全、更容易推理，但也意味着第一版统计在来源系统口径不一致时，可能需要标记为部分、估算或冲突。

本架构优先保证长期可信和本地稳定，而不是追求激进的自动集成。
