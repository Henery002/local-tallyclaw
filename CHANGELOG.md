# 更新日志

本文件按日期降序简要记录 TallyClaw 的功能迭代、问题修复、文档调整、架构决策和重要安全边界变化。

## 记录规则

- 最新日期放在最上方。
- 同一天可以合并记录，但需要按类别分组。
- 功能迭代、问题修复、重要架构调整、安全边界变化和用户可见变更应记录。
- 普通讨论、阶段性探索、临时设计资源和低价值过程信息不需要记录。
- 记录应简明说明“发生了什么”，必要时附相关文档路径。

常用类别：

- `新增`：新的能力、文档或目录结构。
- `变更`：对既有行为、方案或规范的实质调整。
- `修复`：问题修复。
- `安全`：安全、隐私、凭据、只读边界相关变更。
- `文档`：文档结构、维护规则或说明调整。

## 2026-05-12

### 新增

- 窗口趋势卡片新增最近 6 小时 token 消耗曲线：按 30 分钟桶展示 TallyClaw 已观测到的持久化增量，曲线位于 7 日自然日柱状图上方，并支持 hover 查看半小时 token 信息。

### 变更

- 调整展开面板信息顺序：`7 天成功率` 下移到 `来源健康` 上方，继续使用 7 天滚动请求窗口口径展示成功率、请求数、失败数和平均延迟。
- 优化最近 6 小时曲线交互：标题标注 `30m` 粒度，空闲时显示当前半小时滚动累计，最新点加入轻量脉冲，方便观察正在变化的实时桶。
- 优化 activity 态宠物细节：将活跃眼睛从尖号改为更克制的专注扫描猫眼，并把粗胶囊核心动效收敛为细能量条，避免活跃态表情显得笨重。
- 优化常驻性能：宠物根视图从固定 30fps Timeline 改为自控轻量时钟，idle / warning / 收起态不再持续 tick，hover 时升至 12fps，真实 activity 时降为 2fps，粒子/按压短动画才临时使用 30fps；展开面板改为仅展开时挂载，避免隐藏面板继续参与刷新。
- 优化刷新尖峰：四个上游数据源的只读 snapshot 读取改为并发执行，ledger 记录、防回退和 exact 事件写入仍保持顺序处理，降低慢来源拖住整轮刷新的概率。

### 修复

- 优化总计说明口径：展开面板改为说明“总计含已读到的上游历史累计”，并优先展示各上游可读到的最早累计/事件起点；读不到上游起点时才降级为未知，同时继续说明 TallyClaw 自有 ledger 会持久化防回退，避免把 ledger 写入日期误解为 token 产生日期。
- 调整总计起点展示优先级：本机自用阶段 cockpit / Codex 是 token 大头，展开面板优先显示 cockpit 可读到的累计起点；cockpit 起点读不到时再回退到其他上游起点，避免 OpenClaw/Hermes 早期零散历史抢占说明口径。
- 增强面板观测值永久化：SQLite ledger 为每个来源/窗口同时保存“展示累计值”和“上一轮原始上游快照”，当上游统计文件被清理或重置后，今日、7 天、30 天、总计、请求数、成功率和 7 日柱状图不会回退；后续新产生的用量会按 reset 后 delta 继续累加到已观测事实上。
- 增强 exact 事件持久化：事件指纹加入 observedAt，避免上游库重建后复用事件 ID 覆盖旧事件；同时兼容旧指纹格式，避免升级后重复插入已记录的同一事件。
- 修复当前窗口统计被上游会话清理拖低的问题：今日、7 天、30 天在同一来源和同一窗口 key 内也采用已记录高水位，避免已观测过的 token 因本地会话文件删除而清零或减少；新窗口自然切换时重新开始记录。
- 优化 hover 小浮层位置：将浮层横向推离宠物本体，降低遮挡脸部的概率，同时保留靠近屏幕边缘时自动换侧以避免裁切。
- 继续收敛点击展开动效回归：宠物本体的微动不再在展开瞬间归零，贴边缩略态点击展开时也不再叠加 AppKit 窗口位移动画，避免“先探出再变高”的两段式弹跳感；右侧贴边点击展开验证为完整面板落在可见屏幕内。
- 修复窗口状态刷新造成的高 CPU：`FloatingWindowState` 只在窗口 frame / screen / edge attachment 实际变化时发布状态，避免 `FloatingWindowConfigurator` 与 SwiftUI 更新互相触发；同时移除根部 `TimelineView`，防止常驻宠物空闲时被系统动画时钟持续唤醒。
- 修复点击展开时宠物本体弹跳回归：展开动画作用域收敛到面板淡入淡出，宠物舞台显式屏蔽展开事务动画，避免点击面板时本体跟随 SwiftUI 布局动画跳动。
- 继续修复点击展开本体弹跳回归：将窗口展开状态与面板透明度动画拆分，窗口尺寸切换不再进入 SwiftUI 动画事务，仅保留面板自身淡入淡出，避免宠物本体被 resize / layout 动画带动。

## 2026-05-11

### 修复

- 修复边缘吸附缩略态后点击展开面板的动效回归：点击宠物现在只切换展开状态，不再触发边缘吸附的窗口 reveal/restore 位移动画，恢复为宠物本体原地稳定、面板在下方缓动展开的基线交互，同时保留左右贴边与 hover 探出能力。
- 修复本地开发验证时旧 TallyClaw 进程残留的问题：`script/build_and_run.sh` 现在会先尝试正常结束旧进程，若常驻拦截导致进程仍在，再仅对本地调试进程执行 force kill，避免新构建已完成但屏幕上仍是旧窗口/旧逻辑造成误判。
- 修复贴边缩略态点击展开后的半遮掩回归：边缘展开会先把完整面板拉回可见屏幕内，并取消 hover reveal 遗留的延迟收回任务；收起后再恢复贴边 peek。
- 修复点击展开时宠物本体弹跳的问题：点击状态切换不再给整个根视图套全局动画，窗口高度切换也不再使用系统 resize 动画；展开态下宠物位置固定，只保留面板淡入和内部轻微状态动效。

### 变更

- 增加活动强度动效第一步：`UsageActivityMonitor` 会按最近 lifetime token 增量输出 low / medium / high 强度，UI 据此调节 `+tokens` 粒子数量、核心光强和活跃节奏；小增量保持轻反馈，大增量才出现更明显的能量反馈，不改变统计口径、活动状态判定或读取逻辑。
- 增加 cockpit / Codex 活动来源识别与轻量动效：活动监视器会在 token 增量发生时记住本轮主要来源；当来源为 cockpit 时，活跃态宠物面屏出现克制的扫描光，强化 Codex 调用正在产生 token 的体感，但不移动宠物本体、不改变展开/拖拽交互。
- 增加 hover 眼神跟随微动效：鼠标悬停在宠物区域时，idle 状态眼睛会轻微看向鼠标位置，增强贴身助理感；该动效只改变眼睛内部偏移，不影响窗口位置、贴边吸附或点击展开。
- 增强 hover 小浮层信息密度：保留今日 `tokens` 主文案，并在有来源占比时补充主要来源及占比，例如 cockpit / gateway，方便不展开面板也能快速判断当前主要 token 来源。
- 优化展开面板信息结构：成功率栏补充 7 天请求数、失败数和平均延迟；窗口趋势栏新增过去 7 个本地自然日 token 柱状图；事件追溯改为近 7 天 exact 逐请求事件榜单，并明确 cockpit 当前为聚合快照、不进入逐请求榜单，避免陈旧 OpenClaw 历史事件误导。
- 窗口趋势 7 日柱状图支持 hover 信息：鼠标悬停到单日柱图时显示该日标签和 token 数，并高亮当前柱图。
- 成功率栏标题明确为 `7 天成功率`：当前默认口径是 7 天滚动请求窗口，比“今日”更抗小样本抖动，比“总计”更能反映近期稳定性。
- 展开面板改为纵向滚动容器，并统一面板内标题、正文、说明和数值文本字号，降低新增模块后裁切和字号跳动问题。
- 增加今日 digest 轻量解释：展开面板新增今日用量摘要，展示今日 token、相对 7 日均值的高低关系和主要来源；该摘要只基于现有 `UsageSnapshot` 计算，不读取 prompt、不触碰上游数据。
- 增强 cockpit 追溯语义解释：当主要来源是 cockpit 聚合快照、但事件追溯榜单来自 exact 事件来源时，展开面板会明确提示“cockpit 为聚合快照；下方榜单仅含 exact 事件来源”，避免误把 OpenClaw/local-ai-gateway 的事件榜单理解为 cockpit 逐请求明细。
- 增加窗口语义轻量说明：展开面板的窗口趋势卡片补充“今日=本地自然日、7/30 天=滚动窗口、总计=长期累计”的解释，降低今日/7 天/30 天/总计统计口径误读风险。

## 2026-05-10

### 新增

- 增加 `docs/guides/local-roadmap-board.md`，将 TallyClaw 后续推进方向收敛为本机长期自用看板，明确短期不推进对外发布工程，优先推进边缘吸附缩略态、精细追溯面板、事件级账本、性能画像、源适配抗变更和 UI 动效打磨。
- 增加边缘吸附缩略态第一版：收起态拖到左右边缘附近后自动吸附为半露宠物本体，hover 或点击时探出，离开后自动收回，为后续边缘贴身助理交互打基础。
- 增加精细追溯面板第一版：展开面板在四个核心 token 窗口之外，补充窗口趋势、来源健康和主要来源占比，先基于现有 `UsageSnapshot` 提供真实可用的追溯线索，不伪造事件级模型榜单。
- 增加事件级账本最小地基：SQLite ledger 新增 `usage_observations` 观测表，按来源 lifetime 快照生成稳定 fingerprint，重复轮询不会制造重复观测，为后续真实事件级 provider/model 追溯做准备。
- 增加 `local-ai-gateway` 真实事件级观测接入：只读读取 `inference_usage_events` 的稳定事件 ID、来源、provider、model、token 和请求统计，写入 TallyClaw 自有 ledger；ledger 以 exact 事件水位增量读取并聚合 provider/model/sourceName 榜单，重复轮询不会重复记账。
- 展开追溯面板接入事件级榜单：当 ledger 中已有 exact 观测时，优先展示热门模型、provider 和调用来源；没有事件级数据时继续回退到快照来源占比。
- 增加 OpenClaw 与 Hermes 真实事件级观测接入：OpenClaw 复用 legacy ledger key 与 trajectory dedupe key，Hermes 复用 `sessions.id`，并继续排除经本地网关转发的记录，避免与 `local-ai-gateway` 重复统计。
- 增加 cockpit tools 观测接入：由于 cockpit 当前只提供聚合统计 JSON，不具备逐请求事件 ID，TallyClaw 将其写入为 `snapshot` 级观测，补齐 cockpit/codex 来源维度但不将其标记为 exact 事件。
- 增加性能画像与读取节流第一版：来源读取状态记录本次读取耗时并持久化，展开面板的来源健康显示最慢读取耗时；主循环从固定 5 秒轮询改为活跃 5 秒、连续空闲 15/30 秒、读取失败 10 秒的自适应刷新间隔。
- 增加来源 schema 抗变更与健康解释第一版：`local-ai-gateway` 事件读取兼容缺失 `source_event_key` / `upstream_model_id` 的旧 schema；`local-ai-gateway` 与 Hermes 会在核心 SQLite 列缺失时明确报告 schema mismatch 和缺失列名，避免静默算错或只暴露底层 `no such column`。
- 增强 cockpit 与 OpenClaw 格式抗变更：cockpit JSON 缺核心字段时会转成带字段路径的 schema mismatch；OpenClaw legacy ledger 会跳过单条缺 `provider` 的 malformed 事件，保留同一文件内其他有效记录。

### 变更

- 调整本机自用阶段后续优先级：当前重点转为 cockpit / Codex 侧 token 观测准确性、解释性和交互反馈；OpenClaw 作为第二优先级，local-ai-gateway 当前停服但保留兼容和后续恢复空间，Hermes 暂不作为近期主线。

### 修复

- 彻底修复宠物拖不到屏幕顶部、顶部残留大块空白的问题：根因是收起态仍使用 308x420 的透明 SwiftUI 窗口，macOS 约束的是整个透明窗口而不是可见宠物，导致宠物被窗口内部空白拖住。现在收起态窗口改为 308x110，宠物舞台贴齐窗口顶部，展开态按需切换到 308x300 且保持窗口顶边不下移；历史保存过的 420pt 窗口高度会按顶边位置迁移到新尺寸，避免旧偏好继续复发。
- 移除不稳定的运行时 `object_setClass` / `UnconstrainedWindow` 方案，避免 SwiftUI 创建的 `NSWindow` 被换类后触发 KVO 崩溃。
- 优化宠物左右贴边拖拽与展开交互：左右夹边目标从 170pt 宠物舞台改为 64pt 可见宠物本体，使收起态可贴近屏幕左右边缘；点击展开改为窗口从顶部锚定缓动变高，面板在宠物下方原地淡入，宠物本体不再跟随缩放弹起。

## 2026-05-09

### 新增

- 增加对 OpenClaw `.trajectory.jsonl` 会话轨迹文件的读取支持，以兼容最新的 OpenClaw 用量记录机制。现已能够全面统计所有 Agent（包括 main、ops 等）和所有交互通道（Web 端、飞书 DM、企业微信群、Cron 任务）产生的会话 token，并自动排除已由 local-ai-gateway 代理的模型（如 openai）以避免重复计算。
  - **热修复**：修正了解析逻辑中因为复用 `traceId` 与 `seq` 导致同一个 Session 内的多次运行（多轮对话）数据被误伤去重的问题，已引入 `runId` 确保单次模型生成事件的绝对唯一。
  - **实时性优化**：通过额外解析 `.trajectory.jsonl` 中的 `prompt.submitted` 提词提交事件，成功实现了用量监视器的“0延迟启动”响应；同时将原本为兼容 Cockpit 长轮询而设置的 30s+15s 活跃尾盘激进缩减至 8s+8s，彻底解决了由于等待超时导致的宠物“延迟停滞”现象。
  - **UI 与动效升级**：补齐了 Figma 原设计稿中的待机眨眼动效，并引入平滑的伪随机频次算法；重绘了活跃状态下的小嘴巴（从圆点奶嘴优化为具有能量律动感的伸缩胶囊）；新增了拟真生物感的随机“抖耳”交互（闲置和活跃时均有几率独立抖动左右耳），并运用二阶贝塞尔曲线（QuadCurve）将原先僵硬的三角形重绘为自然流畅的仿生猫耳，让桌面宠物彻底告别呆滞贴纸感。

### 修复

- 修复开机自启后出现 2 个菜单栏图标且宠物不显示的问题：`WindowGroup` 替换为 `Window` 单实例 scene，增加 `disableRelaunchOnLogin()` 防止系统双重启动，延迟激活窗口确保 scene 实例化完成，`toggleMainWindow` 精准定位主宠物窗口。
- 修复宠物无法被拖拽至屏幕顶部（遗留大量空白）的问题：修正了 `FloatingWindowState` 拖拽碰撞检测中的坐标系计算错误，在 macOS 上将 SwiftUI `GeometryProxy` 的左上角坐标系统正确转换为 AppKit 窗口拖拽所需的左下角相对距离，并使用 `visibleFrame` 适配避让菜单栏。
- 进一步定位宠物拖拽顶部仍残留大空白的问题：真实根因是 420pt 透明 SwiftUI 浮窗被系统约束在菜单栏下方，且运行时 `GeometryProxy` 对宠物区域偶发回传零尺寸，导致拖拽只能按窗口中心夹边；后续最终修复见 2026-05-10 记录。
- 修复「今日」token 显示未在 0 时重置的问题：cockpit tools 的 `daily` 是滚动 24h 窗口而非日历天，增加 `validatedPeriodStats()` 校验每个窗口的 `since` 时间戳，跨越日历边界的窗口返回空值避免膨胀；同时清理 ledger 中已污染的 today 记录。
- 修复当前窗口 token 统计偏低/偏高的问题：cockpit 的 7 天/30 天滚动窗口改为按上游 `updatedAt` 校验，避免因本地读取延迟被误判过期；ledger 仅对总计保留“只增不减”，今日/7 天/30 天允许随滚动窗口回落，并按 TallyClaw 读取时间写入当前窗口，避免旧周/月桶污染当前展示。
- 修复部分来源无法给出日历日窗口时「今日」长期为 0 的问题：当上游只提供滚动窗口但 lifetime 发生正向增长时，TallyClaw 用自有 ledger 将增量补入当前今日/7 天/30 天窗口，不回写上游。
- 修复宠物 activity 误触发问题：状态切换只由主展示 token 总量增长触发，不再因 request 计数或零 token 提交事件单独进入活跃态；同时取消生产 UI 启动时用 preview 示例值预热活动监视器，避免“示例数据 -> 真实 ledger”被误判为新增用量。
- 修复宠物拖拽顶部限制的残余问题：将夹边目标从透明窗口/中心点改为宠物本体矩形，允许 420px 透明窗口按需越界，同时保证宠物本体能贴近可见屏幕边缘；移除遗留拖拽 debug 输出。
- 将 hover 小浮层和 token 粒子的 `tk` 文案改为 `tokens`。
- 修复宠物在活跃期间短暂回落 idle 的抖动问题：活跃持续时间从 6s 增至 30s，增加 15s 渐进冷却扩展，`activeUntil` 只延长不收缩，覆盖模型交互中工具调用和文件写入等自然暂停。

## 2026-05-08

### 新增

- 增加 `SQLiteLedgerStore` 最小可用实现，使用项目自有 SQLite 存储长期 token 统计。
- 增加 `TallyClawLedgerTests`，覆盖重复快照去重、跨来源聚合、上游 reset 后永久总量不倒退和数据库重开读取。
- 初始化 SwiftPM macOS 工程，包含 `TallyClaw` 可执行 target 与 `TallyClawCore`、`TallyClawDataSources`、`TallyClawLedger`、`TallyClawUI` 模块。
- 增加最小可运行 SwiftUI/AppKit 桌面宠物浮窗骨架。
- 增加核心用量模型、只读数据源协议、内存账本协议和 Core 测试。
- 增加 `script/build_and_run.sh` 与 Codex Run action 配置，支持后续一键构建运行。
- 增加 cockpit tools 与 `local-ai-gateway` 首轮只读数据源适配器，以及 fixture 驱动测试。
- 增加多来源 `UsageSnapshot` 合并逻辑，并让 app 启动后读取真实本地统计来源。
- 创建 TallyClaw 初始仓库骨架。
- 增加首版技术方案，明确 TallyClaw 是只读、本地、可持久化的 token 可观测宠物。
- 增加首版 TallyClaw 宠物 UI 概念预览图，存放于 `assets/previews/`。

### 变更

- 将 app 数据路径从“直接展示只读源临时合并快照”调整为“只读源写入 TallyClaw 自有 ledger，再由 UI 读取 ledger 聚合结果”。
- 将首版 SwiftUI 宠物浮窗 UI 替换为 Figma Version 18 设计翻译版，包含独立宠物本体、状态色、idle/high/warning 动效、token 粒子、hover 数据叶片和 click 展开数据条。
- 将应用窗口调整为更小的透明无边框浮窗，并隐藏传统窗口按钮，弱化 widget / dashboard 感。
- 微调宠物浮窗 UI：移除宠物外圈背景、降低 idle/高活跃状态动效幅度、缩小宠物本体、修复 hover 数据叶片与底部展开面板裁切，并让点击面板从宠物下方展开且显示中文观测数据。
- 增加真实用量活动监视器，宠物 activity 状态改为由 lifetime token / request 增量触发，并在短暂活跃窗口后自动回落 idle，避免仅因已有历史用量长期保持 activity。
- 校准快照健康状态语义：成功读取的静态快照标记为 idle，读取异常才标记 warning；同时记录每个数据源的 available / missing / failed 状态、最近读取时间和错误摘要。
- ledger 增加 `source_read_statuses` 持久化表，保存每个来源最近一次读取状态，应用重启后仍可恢复 source 健康上下文。
- 新增 `OpenClawUsageDataSource` 与 `HermesUsageDataSource`，并默认排除经 `local-ai-gateway` (`127.0.0.1:8787`) 转发的流量，避免与网关事件表重复统计。
- 去重规则扩展到 `cockpit` 本地 API 中转端口 `127.0.0.1:56267`；同时主展示总量不再将 cache token 计入，避免“今日/总计”显著偏高。
- 将 `local-ai-gateway`、`openclaw`、`hermes` 的 7 天 / 30 天窗口统一修正为过去 `7 x 24h` / `30 x 24h`；今日维持当天 0 点到现在。
- 点击展开面板改为两排四项：`今日`、`7 天`、`30 天`、`总计`。
- 将数据刷新从启动时单次读取调整为 5 秒准实时轮询，以便 UI 后续反映 token 增长。
- 增加浮窗桌面生命周期偏好：拖拽后位置持久化、常驻顶层/普通浮窗模式切换、开机启动开关。
- 增加系统菜单栏入口，便于直接切换常驻顶层、开机启动、打开设置或退出应用。
- 应用改为系统级常驻形态：启动后不在 Dock 保留图标，并拦截终止请求为隐藏主界面，保留菜单栏入口常驻。
- 菜单栏交互优化为单一“显示/隐藏主界面”切换按钮，并恢复“退出 TallyClaw”真实退出能力。
- 将存量项目文档统一转换为中文，并明确后续文档默认以中文维护。
- 将文档维护规则收敛为轻量模式：默认只维护 `CHANGELOG.md`，停止继续维护 `docs/records/` 类过程记录目录。
