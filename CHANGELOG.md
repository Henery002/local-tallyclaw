# 更新日志

本文件按日期降序记录 TallyClaw 的功能迭代、问题修复、文档调整、架构决策和重要运维状态变化。

## 记录规则

- 最新日期放在最上方。
- 同一天可以合并记录，但需要按类别分组。
- 每次功能迭代、重大讨论、问题修复、架构调整、安全边界变化或 Agent 运维状态变化都应记录。
- 记录应简明说明“发生了什么”，必要时附相关文档路径。

常用类别：

- `新增`：新的能力、文档或目录结构。
- `变更`：对既有行为、方案或规范的实质调整。
- `修复`：问题修复。
- `安全`：安全、隐私、凭据、只读边界相关变更。
- `记录`：讨论归档、决策记录、交接与回溯信息。
